Factory Inventory Management System

System architecture overview — Vue 3 frontend, FastAPI backend, in-memory mock data

Tech Stack

Frontend

Vue 3

Composition API · Vite · Port 3000
  • vue-router for client-side routing
  • axios for HTTP requests
  • Composables for shared state (filters, auth, i18n)
  • Scoped component styles, custom SVG charts
Backend

FastAPI

Python · Uvicorn · Port 8001
  • Pydantic models for request/response validation
  • REST endpoints under /api/*
  • In-memory filtering (warehouse, category, status, month)
  • CORS enabled for local dev
Data

Mock JSON

server/data/*.json
  • No database — flat JSON files
  • Loaded into memory via mock_data.py at startup
  • Covers inventory, orders, backlog, demand, spending
  • Changes are not persisted between restarts

Request / Data Flow

1

User interacts with a view

e.g. selects a filter on Dashboard.vue or Inventory.vue, updating shared refs in the useFilters composable.

2

Component calls the API client

The view calls a method on client/src/api.js (e.g. api.getInventory(filters)), which builds query params and fires an axios request.

3

FastAPI receives and filters

server/main.py routes the request to an endpoint, which applies shared filter helpers (apply_filters, filter_by_month) over in-memory lists.

4

Response validated by Pydantic

Results are shaped against a response model (e.g. InventoryItem, Order) before being serialized back as JSON.

5

Vue stores and derives state

The component stores the raw response in a ref (e.g. allOrders) and computed properties derive display-ready values from it.

6

UI re-renders reactively

Vue's reactivity system updates the DOM — tables, SVG charts, and summary cards — without manual DOM manipulation.
